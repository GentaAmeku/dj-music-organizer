# ADR 0011: BPM精度改善は内部解析を主系統にする

## Status

Accepted

## Context

前回の方針では、rekordboxで解析したBPMを補正データとして取り込めるようにした。

しかし、このプロジェクトの当面のゴールは、外部ツールの値に合わせることではない。
まずはこのプログラム単体で、どこまで正確なBPM推定ができるかを確認し、改善することである。

また、BPMには半速/倍速の曖昧性がある。
外部データで上書きすると、プログラムがなぜ間違えたのかを学びにくくなる。

## Decision

通常の整理処理では、外部BPM補正を使わない。

`bpm_overrides.json` が存在しても、`--use-bpm-overrides` が明示されない限り `dj_music_organizer.py` は補正値を使わない。

BPM推定の既定ポリシーは `consensus` とする。
`consensus` は以下の内部解析結果を組み合わせる。

- 全体音源のオンセット解析
- 打楽器成分のオンセット解析
- beat tracking
- tempogram
- 隣接オンセット間隔

半速/倍速の曖昧性は、内部解析に十分な根拠がある場合だけ解消する。
たとえば、99 BPMと198 BPMが競合し、198 BPMにオンセット間隔またはtempogramの直接支持がある場合、198 BPMを選べる。

rekordbox取り込みツールは、比較、検証、手動補正の補助として残す。
ただし、精度改善の主系統ではない。

## Consequences

- BPM推定の失敗原因をプログラム側で追いやすくなる。
- 外部ツールに依存せず、通常運用の再現性が上がる。
- 半速/倍速問題に対して、根拠付きの自動選択ができる。
- それでもBPM推定は完全ではないため、ログの `bpm_alternatives` と `bpm_confidence` を確認する運用が必要になる。
