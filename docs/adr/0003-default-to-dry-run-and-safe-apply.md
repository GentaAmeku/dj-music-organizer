# ADR 0003: 通常実行は dry-run にする

## Status

Accepted

## Context

このツールは音源ファイルのコピー、リネーム、移動を行う。

BPMとKeyは推定値であり、解析ミスや命名ミスが発生し得る。特にKey推定は不安定な場合がある。

## Decision

通常実行は dry-run とする。

`--apply` を指定した場合も、変更予定を表示して `Proceed? [y/N]` を確認する。

確認なしで実行したい場合のみ `--apply --yes` を使う。

`--apply` は以下を行う。

- `01_Analyzed` に整理済み成果物をコピーする
- 元ファイルを `90_Archive` に移動する
- 上書きはしない
- 失敗したファイルがあっても全体処理を止めない

## Consequences

誤操作で音源を失うリスクを下げられる。

実運用では dry-run を確認してから apply する一手間が増える。
