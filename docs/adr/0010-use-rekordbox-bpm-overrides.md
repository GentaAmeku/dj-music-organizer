# ADR 0010: rekordbox BPMを補正値として扱う

## Status

Superseded by ADR 0011

## Context

`librosa` などの音声解析によるBPM推定は、DJ用途のBPMと大きくずれることがある。

特に以下のケースでズレやすい。

- 半分または倍のテンポで検出される
- イントロ、ブレイク、アウトロのテンポ感が曲本体と異なる
- ゲーム音源のように展開が多く、拍の強さが一定でない

一方で、ユーザーはrekordboxで解析したBPMを目視確認できる。
この値はDJ運用上の実用値として扱える。

## Decision

BPM解析器の結果は推定値として扱う。

既定の整理処理では、解析器が選んだBPMをそのまま使う。
半分/倍の候補は `bpm_candidates` としてログに残すが、物理フォルダやファイル名を黙って補正しない。

rekordboxで確認したBPMは `bpm_overrides.json` に取り込み、明示的な補正値として使う。

補正値が一致した場合、今後の整理では補正値を解析結果より優先する。
既に `01_Analyzed` にある曲は、補正ツールでプレビュー後に移動・リネームできる。

補正ツールは `bpm_corrected` ログを追記し、プレイリスト生成が最新の移動先を参照できるようにする。

## Consequences

- rekordboxとの差分を学習データとして蓄積できる。
- 自動的な半分/倍補正による誤配置を避けられる。
- BPM修正後も、コレクション別BPMプレイリストを再生成できる。
- rekordboxエクスポートを取り込む追加手順が必要になる。
